# [PART 1] IT 컨설팅 리포트 (STEP 1 ~ 3)
## STEP 1. 프로젝트 상세 기획 (Project Planning)
**1. 프로젝트 개요**
*   **Project Name:** 기업 통근버스 통합 배차 및 실시간 관제 시스템 (FMS) 구축
*   **Core Value:** 복잡한 수기 배차 업무의 자동화 및 실시간 운행 모니터링을 통한 '정시성 확보'와 '운행 이슈(결행/지연/조기출발) 즉각 대응'.
*   **Target:** 운송 담당 관리자(Admin), 승무원(Driver), (선택적) 탑승 임직원.
**2. 시스템 아키텍처 & 기술 스택 제안**
*   **Web Admin (관리자용):**
    *   **Frontend:** React.js 또는 Vue.js (실시간 관제 지도 및 복잡한 배차 테이블 처리에 유리).
    *   **Backend:** Java (Spring Boot) 또는 Node.js (NestJS). 실시간 위치 데이터 처리를 위해 WebSocket/MQTT 프로토콜 필수 적용.
    *   **Map API:** Naver Maps 또는 Kakao Maps API (국내 도로 환경 최적화).
*   **Mobile App (승무원용):**
    *   **Platform:** Android (전용 단말기 또는 기사님 개인폰 활용). Flutter 또는 Native(Kotlin) 권장.
    *   **Core Function:** GPS 백그라운드 전송, 운행 시작/종료 처리, 배차 스케줄 확인, 푸시 알림 수신.
*   **Anomaly Detection Engine (이상 감지):**
    *   **Logic:** 정류장별 도착 예정 시간(ETA) vs 실제 GPS 도달 시간 비교 알고리즘.
    *   **Alert:** 오차 범위(예: ±5분) 초과 시 관리자 웹 및 앱으로 즉시 알림 발송.
**3. 기능 명세 (Feature List)**
*   **관리자 (Web):**
    *   **기초 정보:** 승무원(기사) 등록, 차량 정보(차종, 번호, 정원) 등록, 노선/정류장 관리.
    *   **배차 관리:** 일별/월별 배차 계획 수립, 엑셀 일괄 업로드, 긴급 배차 수정.
    *   **실시간 관제:** 지도 위 차량 이동 경로 표시, 현재 속도 및 상태 모니터링.
    *   **이상 징후 알림 (핵심):**
        *   **조기 출발:** 정해진 시간보다 먼저 정류장을 이탈한 경우.
        *   **지연 출발/도착:** 교통 체증 등으로 예정 시간보다 늦어지는 경우.
        *   **결행:** 배차된 차량이 운행 시작 신호를 보내지 않거나 경로를 이탈한 경우.
    *   **통계/리포트:** 운행 일지 자동 생성, 지연/결행 이력 리포트.
*   **승무원 (App):**
    *   로그인 및 배차 스케줄 확인.
    *   운행 시작/종료 버튼 (GPS 트래킹 트리거).
    *   공지사항 수신 및 비상 상황 보고.

## STEP 2. 유형별 비교 견적 및 상세 산출 근거 (Detailed Estimation)
### **TYPE A: 전문 SI / 관제 솔루션 커스텀 (High Stability)**
*   **분석:** UVIS와 같은 전문 관제 시스템 수준의 안정성과 정밀한 GPS 보정 기술(터널, 음영지역 처리)을 적용. DTG(디지털 운행 기록계) 하드웨어 연동까지 고려 가능한 스펙.
*   **[상세 산출 근거]**
    *   **투입 인력 (4.0개월):**
        *   PM (특급): 1.0 M/M x 4.0 (전체 총괄 및 하드웨어 연동 설계)
        *   PL/Backend (특급): 1.0 M/M x 4.0 (대용량 트래픽 처리, 관제 서버)
        *   Frontend (고급): 1.0 M/M x 3.5 (관제 대시보드)
        *   Android Dev (고급): 1.0 M/M x 3.0 (GPS 백그라운드 최적화)
        *   Designer (중급): 0.5 M/M x 2.0
    *   **총 공수:** 약 15.5 M/M
    *   **단가:** SW기술자 노임단가 100% + 제경비/이윤 + 지도 API 비용 별도.
*   :돈가방: **예상 견적 범위: 12,000만 원 ~ 15,000만 원**
### **TYPE B: 중소형 개발사 / 모바일 GPS 기반 (Cost-Effective)**
*   **분석:** 별도의 하드웨어(DTG) 연동 없이, 기사님 스마트폰의 GPS만을 활용하여 구축. '빅픽쳐2'와 유사한 소프트웨어 중심의 배차/관제 기능 구현에 집중.
*   **[상세 산출 근거]**
    *   **투입 인력 (3.0개월):**
        *   PM 겸 개발팀장 (고급): 1.0 M/M x 3.0
        *   Web/Server 개발자 (중급): 1.0 M/M x 3.0
        *   App 개발자 (중급): 1.0 M/M x 2.5
        *   디자이너 (초급): 1.0 M/M x 1.0
    *   **총 공수:** 약 9.5 M/M
    *   **단가:** 합리적인 맨먼스 단가 적용.
*   :돈가방: **예상 견적 범위: 6,000만 원 ~ 8,000만 원**
### **TYPE C: AI 활용 / 기본 기능 중심 (MVP)**
*   **분석:** 복잡한 관제보다는 '배차 스케줄링'과 '단순 위치 확인'에 초점. 이상 감지 로직을 단순화하여 개발 비용 절감.
*   **[상세 산출 근거]**
    *   **투입 인력 (2.5개월):**
        *   풀스택 개발자 (고급): 1.0 M/M x 2.5 (AI 코딩 도구 활용)
        *   앱 개발자 (중급): 0.5 M/M x 2.5
    *   **생산성 혁신:** Admin 페이지의 CRUD 및 기본 로직을 AI로 생성하여 공수 30% 절감.
*   :돈가방: **예상 견적 범위: 3,500만 원 ~ 5,000만 원**

[프로젝트 개요]
기업 통근버스 통합 배차 및 관제 시스템 구축

본 프로젝트는 기업 통근버스의 배차 계획부터 실제 운행까지의 전 과정을 시스템화하여 관리 효율을 높이는 것을 목표로 합니다.
특히, 단순 배차 관리를 넘어 실시간 차량 위치를 추적하고 조기 출발, 지연, 결행 등의 운행 이슈를 시스템이 자동으로 감지하여 관리자에게 알림으로써 탑승객의 불편을 최소화하고 운행 정시성을 확보하고자 합니다.
[과업 범위]
1. 시스템 구성
   - 관리자용 웹(Web Admin): 배차, 차량/승무원 관리, 실시간 관제
   - 승무원용 앱(Android App): 운행 스케줄 확인, GPS 위치 전송, 운행 일지
2. 관리자 기능 (Web)
   - 기초 관리: 승무원 정보, 차량 정보, 노선 및 정류장(POI) 등록
   - 배차 관리: 일별/월별 배차 계획 수립, 엑셀 일괄 등록, 배차 수정 및 확정
   - 실시간 관제: 지도(Naver/Kakao Map) 기반 차량 현재 위치 및 상태 모니터링
   - 이상 징후 모니터링: 운행 계획 대비 실제 운행 데이터 비교 분석
   - 통계/리포트: 운행 실적, 지연/결행 이력, 배차 이행률 통계
3. 승무원 기능 (App)
   - 로그인 및 본인 인증
   - 배차 스케줄 조회 (일별/주별)
   - 운행 시작/종료 처리 (GPS 트래킹 활성화/비활성화)
   - 공지사항 확인 및 관리자 메시지 수신
4. 핵심 로직 (이상 징후 감지 및 알림)
   - 조기 출발 감지: 정류장별 설정된 출발 시간보다 일정 시간(예: 3분) 이상 빠르게 이탈 시 경고
   - 지연 출발/도착 감지: 교통 체증 등으로 예정 시간보다 지연 시 알림
   - 결행 감지: 배차된 시간에 운행 시작 신호가 없거나 경로를 완전히 이탈한 경우 알림
   - 알림 방식: 관리자 웹 팝업 및 앱 푸시 알림
