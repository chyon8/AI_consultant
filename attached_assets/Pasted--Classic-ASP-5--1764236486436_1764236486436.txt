1. 프로젝트 분석 및 목표

핵심 과제: 노후화된 기술(Classic ASP, Windows Server)에서 최신 클라우드 네이티브 환경(Linux, Container)으로의 완전한 리빌드(Re-build).

기술적 난이도: 상(High).

이유 1: 이기종 DB 마이그레이션 (MSSQL → MySQL/PostgreSQL) 및 데이터 정합성 보장.

이유 2: 동시접속 5만 명을 수용하기 위한 오토스케일링(Auto-scaling) 및 캐싱 전략 수립.

이유 3: O2O 특성상 위치 기반 서비스와 실시간 채팅, 결제 로직의 유기적 결합.

2. 시스템 아키텍처 & 기술 스택 제안

대규모 트래픽 처리와 유지보수 용이성을 위해 MSA(Microservices Architecture) 지향적 모듈형 모놀리스 구조를 제안합니다.

Frontend (Web): Next.js (React)

이유: 쇼핑몰 및 마켓 정보의 검색 엔진 노출(SEO)이 필수적이므로 SSR(Server Side Rendering)에 최적화된 Next.js가 적합합니다.

Mobile App: Flutter

이유: iOS/Android 단일 코드베이스로 개발 속도 향상 및 유지보수 비용 절감. 네이티브 기능(카메라, 위치, 푸시) 활용 용이.

Backend: Spring Boot (Java/Kotlin) 또는 NestJS (Node.js)

Spring Boot: 대규모 트래픽 처리 레퍼런스가 많고 안정성이 매우 높음 (추천).

NestJS: 빠른 개발 속도와 가벼운 구조, 프론트엔드(React)와 언어 통일성 장점.

Database:

RDBMS: Amazon Aurora (MySQL 호환) - 고가용성 및 읽기 복제본 확장이 용이.

NoSQL/Cache: Redis (세션 관리, 실시간 재고, 랭킹), MongoDB (채팅 로그, 로그 데이터).

Infra: AWS (EKS or ECS, CloudFront, ElastiCache, S3, WAF).

3. 기능 명세 및 데이터 마이그레이션 전략

[DB 마이그레이션]:

기존 ASP/MSSQL 데이터 분석 → 신규 DB 스키마 매핑(Mapping) 설계.

ETL 도구를 활용한 데이터 이관 및 정합성 검증 스크립트 작성.

[핵심 모듈 구조화]:

커머스 모듈: 상품, 주문, 결제(PG), 장바구니, 배송 추적.

O2O/마켓 모듈: 플리마켓 정보, 셀러 모집/승인, 캘린더, 지도 연동.

커뮤니티/채팅: 실시간 채팅(Socket.io/WebSocket), 리뷰, 피드.

라이브커머스(준비): 스트리밍 서버 연동을 위한 인터페이스 설계 (확장성 고려).

[프로젝트 개요]

본 프로젝트는 기존 Classic ASP 기반의 '올픽마켓' 서비스를 최신 기술 스택으로 전면 재구축(Re-build)하는 사업입니다.
단순한 리뉴얼을 넘어, 동시접속 5만 명을 수용할 수 있는 고가용성 아키텍처를 도입하고, 사용자/셀러/마켓장 모두에게 최적화된 UI/UX를 제공하여 플랫폼 경쟁력을 확보하는 것을 목표로 합니다.

[프로젝트 배경 및 목표]

기술 부채 해결: 노후화된 ASP/Windows 환경을 클라우드 네이티브(Linux/Container) 환경으로 전환.

확장성 확보: 라이브커머스 등 향후 기능 확장이 용이한 모듈형 구조 설계.

사용자 경험 개선: 웹-앱 하이브리드 방식의 한계를 극복하기 위해 Cross-Platform(Flutter) 앱 도입.

데이터 자산화: 기존 축적된 데이터를 손실 없이 신규 시스템으로 이관(Migration).

[과업 범위]

1. 시스템 아키텍처 재설계 및 인프라 구축

목표 성능: 동시접속자 50,000명 처리가 가능한 오토스케일링 아키텍처 구현.

인프라: AWS 기반 (EC2/EKS, RDS, ElastiCache, CloudFront 등) 구성.

보안: 개인정보 암호화(SHA-256/AES-256), SSL, WAF 적용, CSRF/CORS 정책 수립.

2. 데이터베이스 마이그레이션

대상: 기존 MSSQL(Classic ASP) → 신규 RDBMS(MySQL/Aurora/PostgreSQL 중 제안).

요건: 회원, 주문, 상품, 셀러 데이터의 정합성 검증 및 무중단(또는 최소 중단) 이관 전략 수립.

3. 플랫폼 개발 (Web & App)

사용자(Front): Next.js 기반 반응형 웹 + Flutter 기반 앱 (Android/iOS).

관리자(Admin): 셀러 심사, 정산 관리, 통계, 배너 관리 등 백오피스 기능 (약 70p).

주요 기능:

O2O 플리마켓 정보 및 지도 연동.

온라인 스토어 (장바구니, 주문, PG 결제, 배송추적).

실시간 채팅 (Socket 통신).

셀러 전용 페이지 (상품 등록, 판매 관리).

4. UI/UX 리뉴얼

최신 트렌드(쿠팡, 당근마켓 등)를 반영한 직관적인 UI/UX 설계.

사용자 목적(구매/판매/정보탐색)에 따른 최적화된 동선 기획.
