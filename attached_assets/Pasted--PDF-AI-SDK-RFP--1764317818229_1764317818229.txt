# [PART 1] IT 컨설팅 리포트 (STEP 1 ~ 3)
## STEP 1. 프로젝트 상세 기획 (Project Planning)
**1. 프로젝트 개요**
*   **Project Name:** 시도교육청 학생 정신상담 앱용 **AI 디지털 휴먼 SDK 및 3D 에셋 구축**
*   **Core Value:** 학생들의 심리적 거부감을 낮추는 **친근한 3D 캐릭터**와, 상담 맥락에 맞춰 실시간으로 반응(표정, 립싱크, 제스처)하는 **고성능 모바일 SDK** 구현.
*   **Target:** 시도교육청 관할 초/중/고 학생 (모바일 앱 사용자).
**2. 시스템 아키텍처 & 기술 스택 제안**
*   **Client Engine (SDK Core):** **Unity 3D** (권장)
    *   *Reason:* 'Unity as a Library (UaaL)' 기술을 통해 기존 Native 앱(Android/iOS)에 3D 뷰를 모듈 형태로 임베딩하기 가장 적합하며, 캐릭터 애니메이션 제어 및 최적화에 유리함.
*   **Rendering Pipeline:** URP (Universal Render Pipeline) - 모바일 저사양 기기 최적화 필수.
*   **AI Integration:**
    *   **Lip-Sync:** Audio-to-Face 미들웨어(Oculus Lipsync, SALSA 등) 또는 자체 알고리즘을 통해 AI 음성(TTS)에 맞춰 입모양 실시간 동기화.
    *   **Emotion Mapping:** LLM API 응답의 감정 태그(예: [Sad], [Happy])를 파싱하여 캐릭터의 Blendshape(표정) 및 Animator(동작) 트리거.
**3. 기능 명세 (Functional Specifications)**
*   **[3D 에셋 모듈]:**
    *   **캐릭터:** 학생 눈높이에 맞춘 캐주얼/세미-실사 스타일 (불쾌한 골짜기 제거), 모바일 최적화 토폴로지(Low-Poly).
    *   **배경:** 심리적 안정을 주는 상담실 (따뜻한 조명 베이크, 정적 오브젝트 위주 최적화).
*   **[애니메이션 모듈]:**
    *   **상담 특화 액션:** 경청(고개 끄덕임, 눈맞춤), 공감(상체 기울임), 위로(손 제스처), 대기(Idle - 자연스러운 호흡).
    *   **페이셜:** 5대 감정(기쁨, 슬픔, 분노, 놀람, 평온) Blendshape 세트.
*   **[SDK 모듈 (Android/iOS)]:**
    *   **View:** Native 앱 내 지정된 영역(View)에 3D 씬 렌더링.
    *   **Interface:** `init()`, `sendText()`, `playVoice()`, `setEmotion()`, `close()` 등 호스트 앱과의 통신 규격 제공.
    *   **Network:** STT/LLM/TTS 서버와의 API 통신 처리 및 응답 대기 상태(Loading) 시각화.

[추천 공고문 제목]
- 시도교육청 학생 정신상담 앱용 AI 캐릭터 3D 에셋 및 모바일 SDK 개발

- 학생 심리 상담을 위한 3D AI 캐릭터 및 기술 플랫폼 구축
[프로젝트 키워드]
- Unity 3D, Mobile SDK, AI Digital Human, Lip-Sync, Character Modeling
[프로젝트 개요]
시도교육청 학생 정신상담 앱 내 AI 캐릭터 및 SDK 모듈 구축
[프로젝트 배경 및 목표]
본 프로젝트는 시도교육청에서 주관하는 학생 정신상담 앱 구축 사업의 일환입니다.
학생들이 거부감 없이 상담에 참여할 수 있도록 친근한 3D AI 캐릭터를 제작하고, 해당 캐릭터가 AI 모델(LLM)의 응답에 따라 실시간으로 반응(말하기, 표정 변화, 제스처)할 수 있는 모바일 SDK를 개발하여 본체 앱(Host App)에 탑재하는 것을 목표로 합니다.
[과업 범위]
1. 3D 콘텐츠 제작
   - 캐릭터 기획: 학생(초/중/고) 눈높이에 맞는 친근하고 신뢰감 있는 상담사 컨셉 도출 (원화 작업 포함)
   - 3D 모델링: 모바일 환경에 최적화된 Low-Poly 모델링, 텍스처링, 리깅 (Rigging)
   - 애니메이션: 상담 특화 모션(경청, 끄덕임, 공감, 위로 등) 및 5대 감정 페이셜 애니메이션
   - 배경 제작: 심리적 안정을 주는 상담실 3D 배경 1종 (모바일 최적화 필수)
2. 모바일 SDK 개발
   - 개발 환경: Unity 3D 기반 (Android/iOS Native 앱 임베딩 지원 필수 - Unity as a Library 등)
   - AI 연동: 텍스트/음성 기반 상담 AI 모델과의 API 연동 인터페이스 구현
   - 인터랙션 구현:
     a. TTS(음성) 데이터에 맞춘 실시간 립싱크(Lip-Sync) 처리
     b. AI 응답의 감정 태그를 분석하여 캐릭터 표정 및 제스처 자동 매핑
   - 최적화: 모바일 저사양 기기에서도 원활한 구동을 위한 렌더링 최적화 (URP 권장)
3. SDK 패키징 및 가이드
   - Android(.aar) 및 iOS(.framework) 라이브러리 형태 제공
   - 호스트 앱 개발자를 위한 연동 가이드 및 샘플 프로젝트 제공
[기술 스택]
- 엔진: Unity 2021 LTS 이상 (권장)
- 언어: C# (Unity), Java/Kotlin (Android Bridge), Obj-C/Swift (iOS Bridge)
- 렌더링: URP (Universal Render Pipeline)
- 통신: REST API, WebSocket (실시간성 필요 시)
[클라이언트 준비 사항]
1. 문서 및 자료
   - 전체 앱 기획서 및 UI/UX 가이드라인 (SDK 영역 정의)
   - AI 모델 API 명세서 (입출력 규격)
2. 협업 환경
   - 본체 앱(Host App) 개발사와의 기술 협의 미팅 주선
[주요 일정]
1. 희망 착수일: 계약 체결 후 즉시
2. 주요 마일스톤
   - 착수 후 3주: 캐릭터 컨셉 확정 및 모델링 초안
   - 착수 후 6주: 애니메이션 제작 및 SDK 프로토타입(연동 테스트)
   - 착수 후 9주: SDK 최적화 및 최종 납품
3. 최종 오픈 희망일: 착수일로부터 약 2.5 ~ 3개월 내외
[지원 자격 및 우대 사항]
1. 지원 자격
   - 3D 캐릭터 모델링부터 리깅, 애니메이션까지 자체 수행 가능한 업체
   - Unity를 활용하여 Android/iOS Native 앱에 모듈(SDK) 형태로 납품한 경험이 있는 업체
2. 우대 사항
   - AI 디지털 휴먼, 가상 비서, 챗봇 아바타 관련 프로젝트 수행 경험
   - 모바일 3D 최적화(Draw call 관리, 텍스처 압축 등) 노하우 보유
   - 교육/공공기관 프로젝트 수행 경험
[산출물]
- 3D 원본 소스 (Maya/Max/Blender 파일 및 FBX)
- Unity 프로젝트 원본 소스
- 빌드된 SDK 파일 (Android/iOS)
- SDK 연동 매뉴얼 및 API 명세서
- 유지보수 가이드
[계약 관련 특이 사항]
본 프로젝트는 'SDK 형태'로 납품되어야 하므로, 단순한 게임 앱 개발이 아닌 **Native 앱과의 통신 및 라이프사이클 관리**에 대한 기술적 이해도가 필수적입니다. 제안서 제출 시 이에 대한 기술적 접근 방안(예: Unity as a Library 활용 방안 등)을 반드시 명시해 주시기 바랍니다.
